# やさしい解説: 色空間変換 (Python 版)

## 色の表し方は 1 つじゃない

色は「赤・緑・青の混ぜ具合」で表すこともできるし、「どんな色合い・どれくらい鮮やか・どれくらい明るい」で表すこともできます。コンピュータには **色の表し方 = 言葉づかい** がいくつかあって、それを **色空間 (color space)** と呼びます。

## 代表的な 4 つ

- **BGR (= RGB の並び替え)**: 赤・緑・青の混ぜ具合。OpenCV のデフォルト。
- **Gray (グレースケール)**: 明るさだけ。色の情報は捨てる。
- **HSV**: 色合い (Hue) / 鮮やかさ (Saturation) / 明度 (Value) の 3 つ。**人間の感じ方に近い**。
- **Lab (L\*a\*b\*)**: 人間の視覚に合わせた均一な空間。色差計算 (ΔE) によく使う。

## なぜ HSV が便利？

たとえば「写真の中から**赤いリンゴ**だけ取り出したい」とき、BGR だと「赤が強くて緑も青も弱い…」と細かく条件を書かないといけません。
でも HSV なら **「色合い (H) が赤の範囲」** と書くだけで済みます。

```python
hsv = cv2.cvtColor(bgr, cv2.COLOR_BGR2HSV)
mask = cv2.inRange(hsv, (0, 80, 80), (10, 255, 255))
```

`cv2.inRange` は条件を満たすマス目を 255、そうでない場所を 0 にした **マスク画像** を返します。NumPy でいえば **ブール配列** の画像版です。

## Python/NumPy で扱う上での注意

### チャネルへのアクセス

```python
b, g, r = cv2.split(bgr)    # OpenCV の関数でチャネル分離
# または
b = bgr[:, :, 0]            # NumPy スライス (こちらが高速)
g = bgr[:, :, 1]
r = bgr[:, :, 2]
```

### 型の注意

`cv2.cvtColor` の結果は `uint8 (0〜255)` のままです。HSV の H チャネルは **0〜180** (360° の半分) なので、一般的な「赤 = 0°」を確認するときは値を半分にして考えます。

## 一言まとめ

**「同じ色を、別の言葉で言いかえる」** のが色空間変換。NumPy 配列として扱えるので、変換後は `img[:,:,0]` のようにチャネルを取り出してそのまま計算できます。
