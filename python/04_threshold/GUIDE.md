# やさしい解説: 閾値処理 (二値化) (Python 版)

## 二値化ってなに？

画像のマス目 1 つひとつを見て、

- 明るい (= ある値より大きい) → **白 (255)**
- 暗い (= ある値より小さい) → **黒 (0)**

の 2 種類だけにする処理です。グレーの世界が **白黒だけの世界** になります。

## なぜ二値化するの？

「文字だけ取り出したい」「物の形だけ見たい」というとき、色や微妙な明るさはむしろジャマです。**白か黒か** にしてしまえば、後の処理 (輪郭を取るなど) がとても楽になります。

身近な例: ノートをスマホでスキャンするアプリは、内部で二値化して「文字 = 黒、背景 = 白」を作っています。

## どこで線を引く？ (= 閾値の決め方)

3 つのやり方を知っておくと便利です。

1. **手動 (固定閾値)**: 「明るさ 127 で区切る」のように自分で決める。
2. **Otsu (大津法)**: ヒストグラムを見てコンピュータが **いちばんよく分かれる値** を自動で選ぶ。
3. **Adaptive (適応的閾値)**: **場所ごとに別の閾値** を使う。照明ムラのある書類に効果的。

## Python での返り値を知っておこう

`cv2.threshold` は **タプルで 2 つの値** を返します。

```python
retval, binary = cv2.threshold(gray, 127, 255, cv2.THRESH_BINARY)
# retval  : 実際に使われた閾値 (Otsu のとき自動計算値が入る)
# binary  : 二値化後の画像 (NumPy 配列)
```

慌てて `binary = cv2.threshold(...)` とだけ書くと、**タプル全体が binary に入ってしまう** ので注意。`_` で不要な返り値を捨てるのが Pythonic です。

```python
_, binary = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY | cv2.THRESH_OTSU)
```

## NumPy との組み合わせ

二値化結果は `uint8` の NumPy 配列なので、そのまま論理演算できます。

```python
combined = mask1 | mask2      # 2つのマスクの OR
inverted = 255 - binary        # ネガポジ反転
```

## こんなときに困る

- 画像にノイズがいっぱい → 二値化したらゴマ塩みたいになる → **先にぼかしてから** 二値化するとマシになる。
- 影が強い → 固定や Otsu だと半分黒くなる → **Adaptive** が出番。

## 一言まとめ

二値化は **「白か黒かに割り切る」処理**。Python では `cv2.threshold` の **返り値がタプル**なことと、結果が **NumPy 配列** なことを覚えれば、あとは自然に使えます。
