# やさしい解説: 閾値処理 (二値化)

## 二値化ってなに？

画像のマス目 1 つひとつを見て、

- 明るい (= ある値より大きい) → **白 (255)**
- 暗い (= ある値より小さい) → **黒 (0)**

の 2 種類だけにする処理です。グレーの世界が **白黒だけの世界** になります。

## なぜ二値化するの？

「文字だけ取り出したい」「物の形だけ見たい」というとき、色や微妙な明るさはむしろジャマです。**白か黒か** にしてしまえば、後の処理 (輪郭を取るなど) がとても楽になります。

身近な例: ノートをスマホでスキャンするアプリは、内部で二値化して「文字 = 黒、背景 = 白」を作っています。

## どこで線を引く？ (= 閾値の決め方)

3 つのやり方を知っておくと便利です。

1. **手動 (固定閾値)**: 「明るさ 127 で区切る」のように自分で決める。簡単だけど画像によってはうまくいかない。
2. **Otsu (大津法)**: コンピュータが **画像全体を見て、いちばんよく分かれる値** を自動で選ぶ。万能ではないけど、よく効く。
3. **Adaptive (適応的閾値)**: **場所ごとに別の閾値** を使う。例: 影のかかった書類のスキャンでは、影の場所と明るい場所で違う値が必要 → これを自動でやってくれる。

## こんなときに困る

- 画像にノイズがいっぱい → 二値化したらゴマ塩みたいになる → **先にぼかしてから** 二値化するとマシになる。
- 影が強い → 固定や Otsu だと半分黒くなる → **Adaptive** が出番。

## 一言まとめ

二値化は **「白か黒かに割り切る」処理**。割り切るラインの決め方を 3 通り (固定 / Otsu / Adaptive) 知っていると、たいていの場面で対応できます。
